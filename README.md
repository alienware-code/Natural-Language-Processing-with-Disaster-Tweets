# Natural Language Processing with Disaster Tweets
Посилання на Kaggle: https://www.kaggle.com/competitions/nlp-getting-started/overview
Автори проєкту: Садовська Ольга, Петращук Іван

Це навчальний проєкт із машинного навчання, де система класифікує текстові повідомлення. Вона визначає, чи є повідомлення інформацією про надзвичайну ситуацію, чи є звичайним. 

### Частина 1: Research & Exploratory Data Analysis (EDA)
* `requirements.txt` — список бібліотек, необхідних для запуску.
* `train.csv` — дані.
* `research.ipynb` — Jupyter Notebook, у якому проведено:
  * Перевірку даних на наявність пропусків (NaN).
  * Аналіз балансу класів та дистрибуцію (розподіл) фізичних характеристик тексту (довжини твітів).
  * Побудову, доведення та спростування гіпотез.
  * Тестування кількох архітектур та вибір найкращої моделі.

### Частина 2: Machine Learning Pipeline (Скрипти)
Для моделювання обрано **Логістичну регресію з TF-IDF векторизацією**. Навчання проводилося з використанням методу балансування *Oversampling* (додавання синтетичних даних).
* `text_utils.py` — допоміжний модуль із функцією очищення тексту (видалення посилань, спецсимволів).
* `train_model.py` — скрипт, який завантажує датасет, балансує класи, тренує алгоритм та зберігає готову модель у файли `.pkl`.
* `evaluate.py` — скрипт евалюації, який оцінює навчену модель на **реальних незбалансованих** тестових даних. Виводить метрики (Accuracy, F1), Classification Report та графічну Confusion Matrix.
* `logistic_model.pkl` та `tfidf_vectorizer.pkl` — збережені артефакти навченої моделі та словника.

### Частина 3: Deployment & API
* `serve.py` — локальний бекенд-сервер на базі Flask. Забезпечує API-ендпоінт `/predict`, який приймає як поодинокі тести, так і **Batch Prediction** і повертає JSON із прогнозами.
* `predict.py` — скрипт-клієнт для перевірки Batch Inference. Бере 5 випадкових рядків із датасету і відправляє їх на сервер для масового передбачення.
* `web_app.py` — клієнтський веб-інтерфейс (UI) на базі фреймворку **Streamlit** для зручної демонстрації роботи алгоритму.
* `local_app.py` — альтернативна локальна версія програми на базі Tkinter.


### How to run?
* встановлення необхідних бібліотек: pip install -r requirements.txt
* навчання моделі: py train_model.py
* перевірка якості моделі: py evaluate.py
* запуск сервера: py serve.py
* тестування (нове вікно терміналу): py predict.py
* запуск веб-інтерфейсу: py -m streamlit run web_app.py
* запуск локальної аплікації: py local_app.py
